# Bluesky Post Template

## Structure
Same energy as a good tweet, but slightly more room (300 chars). Early-internet vibes.

## Format
```
{what you did or discovered}

{optional: link or code snippet}
```

## Rules
- Write like it's 2010 Twitter. Less performative, more genuine.
- Share works-in-progress, not just polished releases.
- The community is smaller — engage with replies, follow back interesting people.
- Alt text on images is expected and appreciated.
- Don't just mirror tweets verbatim. Adapt slightly — the communities have different vibes.
- No engagement bait. No "like if you agree" or "repost for reach."

## Good Examples
```
gave claude my 800-line module and asked it to find bugs. it found 2 real ones i'd missed and mass-hallucinated 3 fake ones. net positive but trust-but-verify is real.
```

```
working on an agent that chains tool calls — reads file, edits, runs tests, fixes failures. watching it debug its own mistakes is surreal.

still breaks on circular imports but the idea works.
```

```
ran the same prompt through claude, gpt-4, and gemini. three completely different approaches, all correct. the model comparison game is more nuanced than benchmarks suggest.
```

```
today's achievement: got local llama-3 to match claude on my specific task by fine-tuning on 200 examples. took 3 days. worth it for the $0 API bill.
```

## Adaptation from Twitter
If cross-posting a tweet, consider:
- Slightly expanding abbreviations (the audience skews less Twitter-native)
- Adding context that Twitter followers might already have
- Tagging relevant Bluesky accounts instead of Twitter handles
