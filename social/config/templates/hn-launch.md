# Hacker News Launch Template

## Title Format
```
Show HN: {Name} – {What it does in under 10 words}
```

No adjectives. No emoji. No exclamation marks. State what it is.

## First Comment (post immediately after submission)
```
Hey HN, I built {name} because {one-sentence problem}.

{2-3 sentences on how it works technically — mention the stack, the approach, interesting tradeoffs}

{What it doesn't do / known limitations — get ahead of criticism}

{1 sentence on what's next}

Repo: {github url}
```

## Rules
- Title is everything. HN moderators will edit clickbait titles.
- Post your first comment immediately — it sets the tone.
- Be honest about limitations. HN respects intellectual honesty.
- Don't ask for upvotes. Don't share the link on Slack asking friends to upvote.
- Submit Tuesday-Thursday, 9-11am ET for best visibility.
- Answer every comment thoughtfully. HN engagement = survival on the front page.
- If someone asks a hard question, say "good question, I haven't thought about that" if you haven't.
- Link to the actual project, not a blog post about the project.

## Good Title
```
Show HN: Mars – CLI for managing multi-repo TypeScript projects
```

## Bad Title
```
Show HN: I built an amazing tool that makes monorepo management effortless and blazing fast
```
