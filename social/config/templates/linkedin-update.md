# LinkedIn Project Update Template

## Structure
Short story about what you shipped, what you learned, or a problem you solved.

## Format
```
[Hook line — the interesting part, visible before "see more"]

[2-3 paragraphs, one sentence per line, telling the story]

[What you learned or what's next]
```

## Rules
- First line is everything. It's all people see in the feed. Make it count.
- One sentence per line. LinkedIn's algorithm and mobile layout reward this.
- No hashtag walls. 3 max, at the very end, only if relevant.
- Don't say "I'm excited" or "proud to share." Just share it.
- Include a specific number or metric if you have one.

## Good Example
```
We deleted 40% of our codebase last week. And everything got faster.

Here's what happened: our CLI tool had accumulated three different config parsers over two years. Each one handled a slightly different format.

Instead of adding a fourth, we spent a week unifying them. One parser, one format, one source of truth.

Build time dropped from 12s to 4s. Test suite from 90s to 35s. And the codebase went from 15k lines to 9k.

Sometimes the best feature is removing features.

#opensource #typescript #devtools
```

## Bad Example
```
🚀 Thrilled to announce that we've released version 2.3 of our innovative tool! This groundbreaking update brings comprehensive improvements to the developer experience. We've leveraged cutting-edge technology to deliver a seamless workflow... [EVERY AI ALARM BELL RINGING]
```
