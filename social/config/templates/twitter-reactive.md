# Twitter Reactive Post Template

## Structure
Quick reaction to something trending. Time-sensitive — draft within hours, not days.

## Types

### Reacting to another tool's release
```
{tool} just shipped {feature}. {your genuine take — what it means, how it affects you, or a witty observation}
```

### Jumping on a trending debate
```
{the debate in 5 words}: {your opinionated take}
```

### Tying your work to trending context
```
everyone's arguing about {trending topic}. meanwhile i just shipped {your thing} which actually {solves/relates to it}.
```

## Rules
- Must add genuine value or humor — never clout-chase an event you have nothing to say about
- React within hours, not days. If it's stale, skip it.
- Your take must be YOUR take. Not a summary of what happened.
- Humor and sarcasm encouraged — this is where witty one-liners shine
- Never dunk on other developers or projects — critique ideas, not people
- If tying to your own work, keep it organic. Forced tie-ins are transparent.

## Good Examples
```
bun just shipped native S3 support. i need to go rethink some things.
```

```
react server components discourse is eternal. we're all just characters in an infinite loop of "but what about the bundle size"
```

```
node just deprecated another API i depend on. annual tradition at this point.
```

```
everyone's talking about AI-generated code. meanwhile i'm over here mass-deleting AI-generated code from a PR. the circle of life.
```

## Bad Examples
```
Congratulations to the Bun team on their amazing release! [EMPTY PRAISE — NO VALUE]
```
```
Big news in the JavaScript ecosystem today! [VAGUE — SAY SOMETHING SPECIFIC]
```
```
Really interesting discussion happening about monorepos right now. [VAGUE — ADD YOUR ACTUAL TAKE]
```
