# Twitter Reactive Post Template

## Structure
Quick reaction to something trending in AI/LLM. Time-sensitive — draft within hours, not days.

## Types

### Reacting to a model launch or AI tool update
```
{company} just shipped {model/feature}. {your genuine take — what it means for your workflow, a witty observation, or what you're going to test first}
```

### Jumping on an AI debate
```
{the debate in 5 words}: {your opinionated take as someone who actually builds with these tools}
```

### Tying your work to trending AI context
```
everyone's arguing about {trending AI topic}. meanwhile i just {built/shipped/broke} something with {tool} that actually {relates to it}.
```

### Reacting to a paper or benchmark
```
new paper: {title/finding}. {your take — does this matter in practice? what are people missing?}
```

## Rules
- Must add genuine value or humor — never clout-chase an event you have nothing to say about
- React within hours, not days. If it's stale, skip it.
- Your take must be YOUR take — as someone who builds with these tools daily, not a summarizer
- Humor and sarcasm encouraged — this is where witty one-liners shine
- Never dunk on other developers or projects — critique ideas, not people
- If tying to your own work, keep it organic. Forced tie-ins are transparent.
- Compare based on real usage, not benchmarks. "I tried X for Y and here's what happened" > "X beats Y on MMLU"

## Good Examples
```
claude just shipped extended thinking. tried it on a codebase refactor — it spent 30 seconds "thinking" and then rewrote 400 lines correctly. terrifying.
```

```
openai dropped codex. first impressions: fast, good at boilerplate, loses context on bigger refactors. claude code still wins for anything over 200 lines imo.
```

```
deepseek-v3 running locally on my M3 at 40 tok/s. we are so back.
```

```
the "AI will replace developers" discourse is eternal. meanwhile i'm using AI to write the boring parts faster so i can spend more time on the interesting parts. different framing.
```

```
new paper claims 95% accuracy on code generation benchmarks. tried the model on actual production code. it forgot to close a database connection. benchmarks remain undefeated.
```

```
gemini just added 2M token context. my entire codebase fits in one prompt now. i don't know if that's exciting or terrifying.
```

```
everyone comparing model scores on HumanEval. nobody comparing which one actually follows your project conventions after the 40th file. that's the real benchmark.
```

## Bad Examples
```
Congratulations to Anthropic on their amazing new model release! [EMPTY PRAISE — NO VALUE]
```
```
Big news in AI today! [VAGUE — SAY SOMETHING SPECIFIC]
```
```
Really interesting discussion happening about AI agents right now. [VAGUE — ADD YOUR ACTUAL TAKE]
```
```
GPT-5 is out! This changes everything! 🚀🔥 [HYPE — SAY WHAT ACTUALLY CHANGED FOR YOU]
```
